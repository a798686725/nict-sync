# NICT 吊数计算器 V2.0 - 局域网同步版

## 概述

NICT 吊数计算器 V2.0 是一款基于 Electron 构建的 Windows 便携版应用，支持通过 SMB 共享文件夹实现局域网内多台电脑实时数据同步。

## 功能特性

- ✅ 便携版 EXE：双击即用，无需安装，无需管理员权限
- ✅ 9 个组别（一组~九组），每组可多页管理船舶数据
- ✅ 实时计算作业时长和效率
- ✅ 深色/浅色主题切换
- ✅ 拖拽排序、填充手柄、Excel 导出
- ✅ **局域网同步**：多台电脑同时编辑，实时同步
- ✅ **冲突处理**：版本冲突时弹出合并对话框
- ✅ **离线模式**：SMB 断开时允许本地继续编辑

## 部署指南

### IT 部门：创建 SMB 共享文件夹

1. 在服务器（或任何可访问的电脑）上创建共享文件夹，例如：
   ```
   \\SERVER\NICT-Calculator
   ```

2. 设置共享权限：所有需要使用的用户都需要**读写权限**

3. 在共享文件夹中创建初始配置文件 `ships.json`（可选，软件首次运行时会自动创建）：

   ```json
   {
     "version": 1,
     "lastModified": "2026-05-15T00:00:00.000Z",
     "groups": {}
   }
   ```

4. 确保所有客户端电脑可以通过网络访问该共享路径

---

### 用户：使用便携版

1. 从 Releases 页面下载 `NICT吊数计算器_v2.0.0_便携版.exe`

2. 双击运行 exe 文件

3. **首次运行**：软件会弹出文件夹选择对话框，请选择 IT 部门提供的 SMB 共享路径（例如 `\\SERVER\NICT-Calculator`）

4. 选择路径后，软件会自动：
   - 在共享文件夹中创建 `ships.json`（如果不存在）
   - 创建 `changelog/` 子文件夹（用于增量日志）
   - 创建 `sync.lock` 文件锁
   - 创建 `version.txt` 版本号文件

5. 其他电脑首次运行时：
   - 同样会弹出文件夹选择对话框
   - 选择**相同的** SMB 共享路径
   - 数据会自动从服务器同步到本地

## 数据同步机制

### 文件结构

```
\\SERVER\NICT-Calculator\
├── ships.json         # 主数据文件（JSON 格式）
├── version.txt        # 当前版本号
├── sync.lock          # 文件锁（防止并发写冲突）
└── changelog/         # 增量日志目录
    ├── change_2026-05-15T08-30-00-000Z_123456.json
    └── change_2026-05-15T08-30-05-000Z_123457.json
```

### 数据格式（ships.json）

```json
{
  "version": 5,
  "lastModified": "2026-05-15T08:30:00.000Z",
  "groups": {
    "group0": {
      "pages": [
        {
          "shipName": "COSCO SHIPPING",
          "estCompletionTime": "2026-05-15T12:00",
          "departureTime": "2026-05-15T14:00",
          "naturalBay": "22-26",
          "longCraneBay": "22-32",
          "qcData": [
            {"bridge": "QC01", "qc": 78},
            {"bridge": "QC02", "qc": 120}
          ],
          "remark": "优先装卸"
        }
      ]
    }
  }
}
```

### 同步原理

1. **轮询机制**：客户端每 2 秒检查服务器上的 `ships.json` 修改时间
2. **文件锁**：写入前创建 `sync.lock`，防止多客户端同时写入
3. **版本号**：每次写入 version++，用于检测冲突
4. **增量日志**：每次变更写入 `changelog/` 目录，便于审计

### 冲突处理

当两台电脑同时修改同一数据时：
- 检测到版本号不匹配，弹出合并对话框
- 用户可选择「保留本地版本」或「使用服务器版本」
- 也可选择「取消」，稍后处理

## 离线模式

如果 SMB 共享暂时不可用（例如网络断开）：
- 软件自动切换到**离线模式**
- 顶部显示红色横幅：`⚠️ 网络断开 · 本地模式`
- 用户可以继续本地编辑
- 恢复连接后，点击状态按钮「🔴 离线」重新连接
- 重新连接后，本地变更会自动同步到服务器

## 配置说明

### 修改 SMB 路径

如果需要更换共享路径：
1. 点击右上角的「🔴 未连接」或「🟢 已连接」按钮
2. 选择「重新选择路径」
3. 选择新的共享文件夹

路径会保存到本地 `localStorage`，下次启动自动连接。

### 清除本地配置

如果需要重新选择路径，可以：
1. 关闭软件
2. 删除 `%APPDATA%\nict-calculator\` 目录（或清除浏览器 localStorage）
3. 重新启动软件

## 常见问题

### Q: 点击 EXE 没反应？

确保解压到了**不含中文的路径**，例如 `C:\Tools\NICT\`。

### Q: 提示"无法连接服务器"？

1. 检查 SMB 共享路径是否正确
2. 检查网络是否畅通（试试在文件管理器地址栏输入共享路径）
3. 确认 IT 部门已为你开放该共享的读写权限

### Q: 多台电脑同时修改会丢数据吗？

不会。软件使用文件锁和版本号机制确保数据完整性。即使同时写入，也只会有一台成功，其他会自动检测到冲突并提示合并。

## 技术架构

- **前端**：HTML + CSS + JavaScript（单文件，无框架）
- **打包**：Electron + electron-builder
- **主进程**：Node.js fs 模块读写 SMB 共享文件
- **IPC**：contextBridge 安全桥接
- **同步**：轮询 + 文件锁 + 版本号乐观锁
